# `unix-sort` - упрощённый аналог утилиты `sort`

Этот проект представляет собой реализацию упрощённого аналога стандартной UNIX-утилиты на языке Go. Программа поддерживает чтение строк из файла или стандартного ввода и вывод их в отсортированном виде с учётом различных режимов сортировки.

Реализация поддерживает все обязательные и дополнительные флаги как GNU `sort`, указанные в задании.

---

## Поддерживаемые флаги

### Обязательные:
- `-k N` - сортировать по N-й колонке (столбцу), разделённой **табуляцией** (нумерация с 1)
- `-n` - сортировать по числовому значению (дробные и отрицательные числа поддерживаются)
- `-r` - сортировать в обратном порядке
- `-u` - выводить только уникальные строки (первая из группы эквивалентных в исходном порядке)
- `-b` - игнорировать начальные и конечные пробелы и табуляции при сравнении

### Дополнительные:
- `-M` - сортировать по названию месяца (`Jan`, `Feb`, ..., `Dec`)
- `-h` - сортировать по человекочитаемым размерам (`1K`, `2M`, `1.5G`, `1Ki`, `2Mi` и т.д.)
- `-c` - проверить, отсортированы ли входные данные; при нарушении вывести ошибку и завершиться с кодом 1

### Файловый ввод/вывод:
- Чтение из файла или `stdin`
- Вывод в `stdout`
- Исходный файл не изменяется

## Особенности реализации

- **Стабильная сортировка**: при равенстве ключей сохраняется исходный порядок строк.
- **Корректная уникальность (`-u`)**: учитывает исходный порядок при выборе первой строки из группы.
- **Полная совместимость флагов**: все комбинации (`-n -r -k 2 -b -u -h -M -c`) работают корректно.
- **Эффективность**: используется `bufio.Scanner` для потокового чтения, сортировка выполняется in-memory.
- **Соответствие GNU `sort`**: поведение проверено на множестве тестов и совпадает с `gsort` (GNU coreutils).
